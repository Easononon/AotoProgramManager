# 品牌差异对照表 — ST语言跨平台对比

> 本表汇总西门子、施耐德、浙中控、横河、和利时在 ST 语言上的语法差异  
> 用于：跨品牌翻译、程序改写、代码转换  
> 最后更新：2026-05-01

---

## 0. 概述对照

| 品牌 | 开发平台 | ST语言名称 | 备注 |
|------|----------|-----------|------|
| 西门子 | TIA Portal | SCL（Structured Control Language）| 本质是ST，语法略有扩展 |
| 施耐德 | Unity Pro / Control Expert | ST | 标准IEC 61131-3 |
| 浙中控 | HolliCSS | ST | 国产DCS |
| 横河 | CENTUM VP | ST/CFC | 日系DCS |
| 和利时 | MACS | ST | 国产DCS |
| 洛克威尔 | Studio 5000 | ST | Logix系列 |

## 1. 变量声明差异

### 1.1 基本声明

| 项目 | 西门子 (SCL) | 施耐德 (ST) | 浙中控 (ST) | 横河 (ST) | 和利时 (ST) |
|------|-------------|------------|------------|----------|------------|
| 全局变量 | `VAR_GLOBAL` | `VAR_GLOBAL` | `VAR_GLOBAL` | `VAR_GLOBAL` | `VAR_GLOBAL` |
| 程序变量 | `VAR` | `VAR` | `VAR` | `VAR` | `VAR` |
| 输入变量 | `VAR_INPUT` | `VAR_INPUT` | `VAR_INPUT` | `VAR_INPUT` | `VAR_INPUT` |
| 输出变量 | `VAR_OUTPUT` | `VAR_OUTPUT` | `VAR_OUTPUT` | `VAR_OUTPUT` | `VAR_OUTPUT` |
| 临时变量 | `VAR_TEMP` | `VAR_TEMP` | `VAR_TEMP` | `VAR_TEMP` | `VAR_TEMP` |

### 1.2 类型定义示例

```st
// 西门子 SCL
VAR_GLOBAL
    MotorSpeed : INT := 0;
    SensorData : ARRAY[1..20] OF REAL;
END_VAR

// 施耐德 ST
VAR_GLOBAL
    MotorSpeed : INT := 0;
    SensorData : ARRAY[1..20] OF REAL;
END_VAR

// 浙中控 ST（类似标准）
VAR_GLOBAL
    MotorSpeed : INT := 0;
    SensorData : ARRAY[1..20] OF REAL;
END_VAR
```

### 1.3 数据类型名称差异

| 标准类型 | 西门子 | 施耐德 | 浙中控 | 横河 |
|----------|--------|--------|--------|------|
| 布尔 | `BOOL` | `BOOL` | `BOOL` | `BOOL` |
| 整型 | `INT` / `DINT` | `INT` / `DINT` | `INT` / `DINT` | `INT` / `DINT` |
| 实数 | `REAL` | `REAL` | `REAL` | `REAL` |
| 字符串 | `STRING(80)` | `STRING(80)` | `STRING(80)` | `STRING(80)` |
| 时间 | `TIME` | `TIME` | `TIME` | `TIME` |
| 日期 | `DATE` | `DATE` | `DATE` | `DATE` |
| 日期时间 | `DATE_AND_TIME` | `DATE_AND_TIME` | `DT` | `DT` |

## 2. 定时器功能块对比

> ⚠️ 各品牌定时器语法基本一致，但实例化方式略有不同

### 2.1 西门子 SCL
```st
// 西门子 TIA Portal
TON_DB(
    IN  := startSignal,
    PT  := T#5s,
    Q   => timerQ,
    ET  => elapsedTime
);
// 或者使用IEC定时器
MyTimer(IN := startSignal, PT := T#5s);
```

### 2.2 施耐德 Unity Pro
```st
// 施耐德
MyTimer TON;
MyTimer(IN := startSignal, PT := T#5s);
TimerOutput := MyTimer.Q;
```

### 2.3 浙中控 HolliCSS
```st
// 浙中控
MyTimer : TON;
MyTimer(IN := startSignal, PT := T#5s);
```

### 2.4 横河 CENTUM VP
```st
// 横河
MyTimer TON;
MyTimer(IN := startSignal, PT := T#5s);
```

## 3. 运算和逻辑差异

### 3.1 逻辑运算符

| 逻辑操作 | IEC标准/西门子 | 施耐德 | 横河 |
|----------|---------------|--------|------|
| 与 | `AND` 或 `&` | `AND` | `AND` |
| 或 | `OR` | `OR` | `OR` |
| 异或 | `XOR` | `XOR` | `XOR` |
| 非 | `NOT` | `NOT` | `NOT` |

### 3.2 字符串操作差异

| 操作 | 西门子 SCL | 施耐德 | 横河 |
|------|-----------|--------|------|
| 连接 | `CONCAT(s1, s2)` | `CONCAT(s1, s2)` | `s1 + s2` |
| 长度 | `LEN(s)` | `LEN(s)` | `STRLEN(s)` |
| 取左 | `LEFT(s, n)` | `LEFT(s, n)` | `SUBSTR(s, 1, n)` |

### 3.3 类型转换

| 转换 | 西门子 | 施耐德 | 横河 |
|------|--------|--------|------|
| INT→REAL | `INT_TO_REAL(i)` | `INT_TO_REAL(i)` | `REAL(i)` |
| REAL→INT | `REAL_TO_INT(r)` | `REAL_TO_INT(r)` | `INT(r)` |
| BOOL→INT | `BOOL_TO_INT(b)` | `b ? 1 : 0` | 不同 |

## 4. 程序结构差异

### 4.1 西门子 SCL 程序结构
```st
// TIA Portal SCL
PROGRAM MainPrg
VAR
    counter : INT := 0;
END_VAR

IF start THEN
    counter := counter + 1;
END_IF;

END_PROGRAM
```

### 4.2 施耐德 ST
```st
PROGRAM MainPrg
VAR
    counter : INT := 0;
END_VAR

IF start THEN
    counter := counter + 1;
END_IF;

END_PROGRAM
```

### 4.3 浙中控 ST
```st
// 浙中控 HolliCSS
PROGRAM MainPrg
VAR
    counter : INT := 0;
END_VAR

IF start THEN
    counter := counter + 1;
END_IF;

END_PROGRAM
// 结构基本一致
```

## 5. 特殊语法扩展

### 5.1 西门子特有
```st
// 西门子 SCL 特有
#变量引用（不用引号）;
"P DB".data := 100;  // 数据块直接寻址
A[0] := A[0] + 1;   // 数组直接访问
```

### 5.2 施耐德特有
```st
// 施耐德 Unity Pro
(* 可以用 AT 指定物理地址 *)
VAR
    Sensor AT %I0.0 : BOOL;  // 直接物理地址映射
END_VAR
```

### 5.3 横河特有
```st
// 横河 CENTUM VP
(* 可能使用不同的关键字风格 *)
(* 部分关键字用大写 *)
IF condition THEN
    value := 100;
END_IF;
```

## 9. 快速翻译参考

### 9.1 西门子 → 施耐德
- 变量声明：几乎相同
- 功能块调用：相同
- 字符串函数：可能需要调整
- 注意西门子的 `"DB".address` 寻址方式在施耐德中不同

### 9.2 西门子 → 浙中控
- 变量声明：相同
- 功能块：相同
- 通讯库：需要替换（Modbus/OPC-UA 库不同）

### 9.3 横河 → 西门子
- 部分关键字不同
- 字符串函数需要调整
- 功能块基本通用
